## ข้อ 1: Refuel (เติมน้ำมัน)

### โจทย์

รถยนต์คันหนึ่งต้องเดินทางจากจุดเริ่มต้น (ตำแหน่ง 0) ไปยังจุดหมายปลายทางที่ระยะทาง $D$ กิโลเมตร โดยเริ่มต้นมีน้ำมัน $F$ ลิตร (วิ่งได้ 1 ลิตรต่อ 1 กิโลเมตร)

ระหว่างทางมีสถานีเติมน้ำมัน $N$ สถานี แต่ละสถานีอยู่ที่ตำแหน่ง $p_i$ และสามารถเติมน้ำมันได้ $f_i$ ลิตร

จงหา **จำนวนครั้งการเติมน้ำมันที่น้อยที่สุด** ที่จะทำให้รถวิ่งถึงจุดหมายได้

### Input

```
D F
N
p_1 f_1
p_2 f_2
...
p_N f_N
```

- บรรทัดแรก: ระยะทางรวม $D$ และน้ำมันเริ่มต้น $F$
- บรรทัดที่สอง: จำนวนสถานี $N$
- $N$ บรรทัดถัดมา: ตำแหน่งสถานี $p_i$ และน้ำมันที่เติมได้ $f_i$

### Output

- จำนวนเต็ม 1 จำนวน คือจำนวนครั้งการเติมน้ำมันน้อยที่สุด
- ถ้าไปไม่ถึง ให้พิมพ์ `-1`

### ตัวอย่าง

**Input:**
```
100 10
4
10 60
20 30
30 20
60 69
```

**Output:**
```
2
```

**คำอธิบาย:**
- เริ่มที่ตำแหน่ง 0 มีน้ำมัน 10 ลิตร
- วิ่งถึงสถานี ตำแหน่ง 10 → เหลือน้ำมัน 0 → เติม 60 ลิตร → มีน้ำมัน 60
- วิ่งถึงสถานี ตำแหน่ง 20 → เหลือน้ำมัน 50 → เติม 30 ลิตร → มีน้ำมัน 80
- วิ่งถึงตำแหน่ง 100 → เหลือน้ำมัน 0 พอดี ✅
- เติมทั้งหมด **2 ครั้ง** (ที่สถานี 10 และ 20)

> 💡 **Hint:** Greedy + Priority Queue (Max Heap) — วิ่งไปเรื่อยๆ พอน้ำมันหมดให้ย้อนกลับไปเติมที่สถานีที่ให้น้ำมันมากที่สุดที่ผ่านมาแล้ว ใช้ max-heap เก็บสถานีที่ผ่าน

## ข้อ 2: Add Machine (เครื่องบวก)

### โจทย์

มีเครื่องจักรที่ทำได้อย่างเดียวคือ **บวกเลข 2 ตัว** โดยถ้าต้องการบวก $a$ กับ $b$ เครื่องจะใช้พลังงาน $(a + b)$ หน่วย

เครื่องสามารถบวกได้เฉพาะตัวเลขที่เคย **add มาแล้ว** (ผลลัพธ์ที่ได้จากการบวกครั้งก่อน) กับตัวเลขใน input

กำหนดตัวเลข $N$ ตัว จงหาลำดับการบวกที่ทำให้ **พลังงานรวมที่ใช้น้อยที่สุด**

### Input

```
N
a_1 a_2 ... a_N
```

### Output

- จำนวนเต็ม 1 จำนวน คือพลังงานรวมน้อยที่สุด

### ตัวอย่าง

**ตัวอย่างที่ 1:**
```
Input:
3
1 2 3

Output:
9
```

**คำอธิบาย:**
- บวก 1 + 2 = 3 → ใช้พลังงาน 3
- บวก 3 + 3 = 6 → ใช้พลังงาน 6
- รวม = 9

ทำไมไม่ทำ (2 + 3) ก่อน?
- บวก 2 + 3 = 5 → ใช้พลังงาน 5
- บวก 1 + 5 = 6 → ใช้พลังงาน 6
- รวม = 11 → **แพงกว่า**

**ตัวอย่างที่ 2:**
```
Input:
4
1 2 3 4

Output:
19
```

**คำอธิบาย:**
- บวก 1 + 2 = 3 → ใช้ 3
- บวก 3 + 3 = 6 → ใช้ 6
- บวก 6 + 4 = 10 → ใช้ 10
- รวม = 19

**ตัวอย่างที่ 3:**
```
Input:
4
4 1 2 3

Output:
19
```

(ลำดับ input ไม่สำคัญ — เลือกลำดับบวกเองได้)

> 💡 **Hint:** Greedy + Min Heap — เหมือนโจทย์ "Huffman Coding" / "Minimum Cost to Merge Stones" หยิบ 2 ตัวที่เล็กที่สุดมาบวกกัน แล้วใส่ผลลัพธ์กลับเข้า heap ทำซ้ำจนเหลือตัวเดียว

## ข้อ 3: Robot (หุ่นยนต์เก็บของ)

### โจทย์

*(เหมือน TOI13 Robot)*

มีตาราง $N \times M$ ประกอบด้วย:
- `X` = ตำแหน่งเริ่มต้นของหุ่นยนต์ (มีได้หลายตัว, สูงสุด 100)
- `E` = พื้นที่ว่าง
- `W` = สิ่งกีดขวาง (เดินผ่านไม่ได้)
- `A` = สิ่งของเป้าหมาย (มีได้หลายชิ้น, สูงสุด 100)

หุ่นยนต์เดินได้ 4 ทิศ (ซ้าย ขวา บน ล่าง) ในแต่ละก้าว

**กฎ:** หุ่นยนต์หยิบของได้ทีละ 1 ชิ้น → ต้องเอากลับไปวางที่จุดเริ่มต้นก่อน → ค่อยไปหยิบชิ้นต่อไป

จงหา:
1. จำนวนสิ่งของ**มากที่สุด**ที่หุ่นยนต์ทั้งหมดเก็บได้สำเร็จ
2. จำนวนก้าว**รวมน้อยที่สุด**ที่ต้องใช้ในการเก็บของให้ได้มากที่สุด

### Input

```
N M
(N บรรทัดของ string ขนาด M ตัวอักษร)
```

### Output

```
(จำนวนของที่เก็บได้) (จำนวนก้าวรวมน้อยที่สุด)
```

### ตัวอย่าง

**ตัวอย่างที่ 1:**
```
Input:
5 5
EEEEE
AEAEW
WEEWW
WEEXE
WWEXA

Output:
3 18
```

**ตัวอย่างที่ 2:**
```
Input:
5 5
WEEEE
AWAEW
WEEWW
WEEXE
WWEEE

Output:
1 6
```

**ตัวอย่างที่ 3:**
```
Input:
5 5
WEEEE
AWEEW
WEEWW
WEEXE
WWEEE

Output:
0 0
```

(ไม่มีทางไปถึงของเป้าหมาย A ได้เลย)

### Constraints

- $1 \le N, M \le 2000$
- จำนวน `X` และ `A` ไม่เกิน 100 ตัว
- Time Limit: 1 วินาที, Memory: 512 MB

> 💡 **Hint:** BFS จากทุกตำแหน่ง X และ A เพื่อหาระยะทางสั้นสุดระหว่างทุกคู่ (X, A) → จากนั้นใช้ **Bipartite Matching** หรือ **Min Cost Matching** เพื่อจับคู่หุ่นยนต์กับของ (ค่า cost = 2 × ระยะทาง เพราะต้องไป-กลับ) ระยะทางหาจาก BFS บน grid

## ข้อ 4: Playlist (จัดการคิวเพลง)

### โจทย์

ให้ implement โครงสร้างข้อมูลเพื่อจัดการ **คิวเพลง (playlist)** โดยรองรับคำสั่งดังนี้:

| คำสั่ง | คำอธิบาย |
|--------|----------|
| `ADD x` | เพิ่มเพลงชื่อ `x` เข้าท้ายคิว |
| `PLAY` | เล่นเพลงแรกสุดในคิว (พิมพ์ชื่อเพลงแล้วลบออกจากคิว) |
| `STATUS` | พิมพ์รายชื่อเพลงทั้งหมดในคิวตอนนี้ |
| `REMOVE k` | ลบเพลงตำแหน่งที่ $k$ ออกจากคิว (1-indexed) |

*(อาจมีคำสั่งเพิ่มเติม เช่น SWAP, SHUFFLE, MOVE)*

### Input

```
Q
(Q บรรทัดของคำสั่ง)
```

### Output

ตอบตามแต่ละคำสั่ง

### ตัวอย่าง

```
Input:
8
ADD Bohemian_Rhapsody
ADD Hotel_California
ADD Stairway_to_Heaven
STATUS
PLAY
REMOVE 1
STATUS
PLAY

Output:
Bohemian_Rhapsody Hotel_California Stairway_to_Heaven
Bohemian_Rhapsody
Stairway_to_Heaven
Stairway_to_Heaven
```

**คำอธิบาย:**
1. ADD 3 เพลง → คิว: [Bohemian_Rhapsody, Hotel_California, Stairway_to_Heaven]
2. STATUS → พิมพ์ทั้ง 3 เพลง
3. PLAY → พิมพ์ Bohemian_Rhapsody แล้วลบออก → คิว: [Hotel_California, Stairway_to_Heaven]
4. REMOVE 1 → ลบตำแหน่งที่ 1 (Hotel_California) → คิว: [Stairway_to_Heaven]
5. STATUS → พิมพ์ Stairway_to_Heaven
6. PLAY → พิมพ์ Stairway_to_Heaven แล้วลบออก → คิว: []

> 💡 **Hint:** Linked List / Doubly Linked List — ใช้ linked list เพื่อให้ลบตำแหน่งกลางๆ ได้เร็ว O(k) หรือถ้าต้องเร็วกว่านั้นอาจใช้ indexed data structure เช่น Order Statistic Tree

## ข้อ 5: Copy (จัดการ String)

### โจทย์

กำหนด string $S$ ความยาว $N$ และมี $Q$ คำสั่ง แต่ละคำสั่งเป็นหนึ่งในสามแบบ:

| คำสั่ง | คำอธิบาย |
|--------|----------|
| `Copy l r` | คัดลอก substring $S[l..r]$ ไว้ใน clipboard |
| `Paste p` | แทรก clipboard ลงที่ตำแหน่ง $p$ ของ string ปัจจุบัน |
| `Find t` | หาตำแหน่งแรกที่ substring/pattern $t$ ปรากฏใน string ปัจจุบัน |

### Input

```
N
S
Q
C_1 S_1
C_2 S_2
...
C_Q S_Q
```

- $N$ = ความยาว string เริ่มต้น
- $S$ = string เริ่มต้น
- $Q$ = จำนวนคำสั่ง
- $C_i$ = คำสั่ง (Copy/Paste/Find)
- $S_i$ = parameter ของคำสั่ง

### Output

ตอบตามคำสั่ง Find — พิมพ์ตำแหน่งที่พบ (1-indexed) หรือ `-1` ถ้าไม่พบ

### ตัวอย่าง

```
Input:
5
hello
4
Copy 1 2
Paste 6
Find he
Find hehe

Output:
1
-1
```

**คำอธิบาย:**
1. String เริ่มต้น: `hello`
2. Copy 1 2 → clipboard = `he`
3. Paste 6 → string กลายเป็น `hellohe`
4. Find `he` → พบที่ตำแหน่ง 1
5. Find `hehe` → ไม่พบ → `-1`

> 💡 **Hint:** String manipulation + Pattern Matching — ถ้า string ใหญ่ อาจต้องใช้ **Rope data structure** สำหรับ Copy/Paste ที่เร็ว และ **KMP** สำหรับ Find ถ้า string เล็ก brute force ก็พอ

## ข้อ 6: Generation (แบ่งเครื่องปั่นไฟ)

### โจทย์

มีเครื่องปั่นไฟ $N$ เครื่อง แต่ละเครื่องมีกำลังผลิต $a_i$ หน่วย

ต้อง **แบ่งเครื่องปั่นไฟออกเป็น 2 กลุ่ม** โดยให้ผลต่างของผลรวมกำลังไฟระหว่าง 2 กลุ่มมีค่า **น้อยที่สุด**

### Input

```
N
a_1 a_2 ... a_N
```

### Output

- ผลต่างน้อยที่สุดที่เป็นไปได้

### ตัวอย่าง

**ตัวอย่างที่ 1:**
```
Input:
3
10 22 5

Output:
7
```

**คำอธิบาย:**
- กลุ่ม 1: {22} = 22
- กลุ่ม 2: {10, 5} = 15
- ผลต่าง = |22 - 15| = 7

**ตัวอย่างที่ 2:**
```
Input:
4
1 2 3 4

Output:
0
```

**คำอธิบาย:**
- กลุ่ม 1: {1, 4} = 5
- กลุ่ม 2: {2, 3} = 5
- ผลต่าง = 0

> 💡 **Hint:** DP Subset Sum — ให้ $S$ = ผลรวมทั้งหมด ปัญหาคือเลือก subset ที่ผลรวมใกล้ $S/2$ มากที่สุด ใช้ DP แบบ knapsack: `dp[j]` = เป็นไปได้ไหมที่จะเลือก subset ให้ผลรวมเป็น $j$? ความซับซ้อน $O(N \times S)$

## ข้อ 7: BFS Level Select (เลือกค่าจาก BFS ทีละชั้น)

### โจทย์

กำหนด Tree ที่มีค่ากำกับที่แต่ละโหนด เมื่อทำ BFS จาก root:
- ในแต่ละ **level** ของ BFS สามารถเลือกได้ **อย่างมาก 1 โหนด**
- จงหา **ผลรวมค่าสูงสุด** ที่ได้จากการเลือก

### Input

```
N
v_1 v_2 ... v_N     (ค่าของแต่ละโหนด)
(N-1 บรรทัด: u v    คือ edge ของ tree)
root
```

### Output

- ผลรวมสูงสุดที่ได้

### ตัวอย่าง

```
Input:
7
3 7 2 9 1 5 8
1 2
1 3
2 4
2 5
3 6
3 7
1

Output:
20
```

**คำอธิบาย:**
```
Tree:
        1(3)
       / \
     2(7)  3(2)
    / \    / \
  4(9) 5(1) 6(5) 7(8)
```
- Level 0: โหนด 1 → เลือก 3
- Level 1: โหนด 2(7), 3(2) → เลือก 7
- Level 2: โหนด 4(9), 5(1), 6(5), 7(8) → เลือก 9 (หรือ 8)
- ผลรวม = 3 + 7 + 9 (หรือ 8) → เลือกดีที่สุด = 3 + 8 + 9 = 20

> 💡 **Hint:** BFS + Greedy ง่ายๆ — BFS level by level แล้วเลือก max ในแต่ละ level รวมกัน

## ข้อ 8: Max Path Sum (เส้นทางผลรวมสูงสุด)

### โจทย์

กำหนด Tree หรือ Graph ที่มีค่ากำกับแต่ละโหนด

จงหา **เส้นทางจาก root ไปยังโหนดใดๆ** ที่มีผลรวมค่าตลอดเส้นทางมากที่สุด

### Input

```
N
v_1 v_2 ... v_N
(N-1 บรรทัด: u v)
root
```

### Output

- ผลรวมสูงสุดของเส้นทางจาก root ไปยัง leaf ใดๆ

### ตัวอย่าง

```
Input:
5
5 3 8 2 10
1 2
1 3
3 4
3 5
1

Output:
23
```

**คำอธิบาย:**
```
      1(5)
     / \
   2(3)  3(8)
         / \
       4(2) 5(10)
```
- เส้นทาง 1→3→5: 5 + 8 + 10 = 23 ✅
- เส้นทาง 1→3→4: 5 + 8 + 2 = 15
- เส้นทาง 1→2: 5 + 3 = 8

> 💡 **Hint:** DFS/BFS จาก root เก็บ sum สะสม — หาค่า max ของ sum ที่ leaf ทุกตัว ง่ายมาก O(N)

## ข้อ 9: BFS with State (BFS เก็บสถานะ)

### โจทย์

กำหนดกราฟ/grid ที่มีค่ากำกับในแต่ละ node ให้ทำ BFS โดยในแต่ละ state ต้องเก็บ:
- ตำแหน่งปัจจุบัน (node)
- ผลรวมสะสม (sum)
- ค่าสูงสุดที่เคยเจอ (max)

และต้องตัดสินใจว่าจะเดินไปทางไหนเพื่อให้ได้คำตอบที่ดีที่สุด

### ตัวอย่างรูปแบบ Input

```
N M    (grid N×M)
(N บรรทัด: ค่าในแต่ละช่อง)
sr sc  (ตำแหน่งเริ่ม)
er ec  (ตำแหน่งเป้าหมาย)
```

### ตัวอย่าง

```
Input:
3 3
1 3 1
1 5 1
4 2 1
1 1
3 3

Output:
(ขึ้นอยู่กับเงื่อนไขที่ต้องการ optimize)
```

> 💡 **Hint:** BFS/Dijkstra + State Space — ทำ BFS แบบ multi-state โดยแต่ละ state คือ `(node, sum, max_so_far)` อาจต้อง prune state ที่ไม่ดีออก ใช้ priority queue ถ้าต้อง optimize

## ข้อ 10: Repeated Substring Pattern (Pattern ซ้ำ)

### โจทย์

กำหนด string $s$ จงตรวจสอบว่า $s$ สามารถสร้างขึ้นได้จากการนำ **substring บางส่วนมาต่อกันซ้ำหลายครั้ง** หรือไม่

กล่าวคือ มี string $t$ ที่สั้นกว่า $s$ โดย $s = t + t + t + ...$ (ต่อ $t$ กัน $k$ ครั้ง, $k \ge 2$)

### Input

```
s
```

### Output

- `true` ถ้า $s$ เป็น repeated substring pattern
- `false` ถ้าไม่ใช่

### ตัวอย่าง

**ตัวอย่างที่ 1:**
```
Input: abab
Output: true
```
(เป็น "ab" ซ้ำ 2 ครั้ง)

**ตัวอย่างที่ 2:**
```
Input: aba
Output: false
```

**ตัวอย่างที่ 3:**
```
Input: abcabcabcabc
Output: true
```
(เป็น "abc" ซ้ำ 4 ครั้ง หรือ "abcabc" ซ้ำ 2 ครั้ง)

### Constraints

- $1 \le |s| \le 10^4$
- $s$ ประกอบด้วยตัวอักษรพิมพ์เล็กภาษาอังกฤษเท่านั้น

> 💡 **Hint:** KMP Failure Function — ถ้า `len % (len - fail[len-1]) == 0` และ `fail[len-1] != 0` แสดงว่าเป็น repeated pattern หรือวิธีง่าย: เช็คว่า `(s + s).find(s, 1) < len(s)` ก็ได้

## ข้อ 11: เลือกของให้ได้มากที่สุดแต่ต่ำกว่างบ

### โจทย์

มีรายการตัวเลือกจัดเป็นกลุ่ม โดยแต่ละกลุ่มมีค่ากำกับ ต้องเลือกจากแต่ละกลุ่มได้อย่างมาก 1 รายการ ให้ผลรวมสูงสุดภายใต้เงื่อนไข (constraint) ที่กำหนด

### Input

```
B G
n_1 → v_1,1 v_1,2 ... v_1,n_1
n_2 → v_2,1 v_2,2 ... v_2,n_2
...
n_G → v_G,1 v_G,2 ... v_G,n_G
```

- $B$ = งบประมาณ (budget) หรือจำนวนเต็มที่กำหนด
- $G$ = จำนวนกลุ่ม

### ตัวอย่าง

```
Input:
10 3
3 → 1 2 3
2 → 2 4
1 → 3

Output:
10
```

**คำอธิบาย:**
- เลือกจากกลุ่ม 1: 3, กลุ่ม 2: 4, กลุ่ม 3: 3
- ผลรวม = 3 + 4 + 3 = 10

> 💡 **Hint:** DP Knapsack หรือ Greedy — ขึ้นกับเงื่อนไข ถ้าเป็น "เลือกจากแต่ละกลุ่ม 1 ชิ้น" คือ **Grouped Knapsack DP** ถ้าเป็นแค่เลือกให้ไม่เกินงบ คือ **0/1 Knapsack** ทั่วไป

## ข้อ 12: เส้นทางสายเปย์ไปหาแฟน (Shortest Path with Budget)

### โจทย์

กำหนดกราฟ $N$ โหนด $M$ เส้นเชื่อม แต่ละเส้นมี **ระยะทาง** $d_i$ และ **ค่าใช้จ่าย** $c_i$

มีงบประมาณ $B$ บาท ต้องเดินทางจากจุด $A$ ไปจุด $B$

**กรณีที่ 1:** ถ้ามีเส้นทางที่ค่าใช้จ่ายรวม $\le B$ → หาเส้นทางที่ **ระยะทางสั้นที่สุด** ภายใต้งบ

**กรณีที่ 2:** ถ้าไม่มีทางไปถึงภายใต้งบ → หาเส้นทางที่ **ค่าใช้จ่ายน้อยที่สุด** แทน (ไม่สนระยะทาง)

### Input

```
N M B
A B
u_1 v_1 d_1 c_1
u_2 v_2 d_2 c_2
...
u_M v_M d_M c_M
```

### Output

- ระยะทางสั้นที่สุด (กรณีมีทางภายในงบ)
- หรือค่าใช้จ่ายน้อยที่สุด (กรณีงบไม่พอ)

### ตัวอย่าง

```
Input:
4 5 10
1 4
1 2 3 5
1 3 2 8
2 4 4 3
3 4 5 1
2 3 1 2

Output:
7
```

**คำอธิบาย:**
- เส้นทาง 1→2→4: ระยะ = 3+4 = 7, ค่าใช้จ่าย = 5+3 = 8 ≤ 10 ✅
- เส้นทาง 1→3→4: ระยะ = 2+5 = 7, ค่าใช้จ่าย = 8+1 = 9 ≤ 10 ✅
- ทั้งสองมีระยะ 7 เท่ากัน → ตอบ 7

> 💡 **Hint:** Modified Dijkstra — state = `(node, cost_used)` ใช้ priority queue จัดเรียงตาม distance เก็บ `dist[node][cost]` = ระยะทางสั้นสุดไปถึง node เมื่อใช้เงินไป cost ถ้าไม่มีทางภายใต้งบ → รัน Dijkstra อีกรอบ optimize ตาม cost แทน

## ข้อ 13: นับคู่หรรษา (Counting Pairs)

### โจทย์

กำหนด array $a$ ขนาด $N$ จงนับจำนวน **คู่** $(i, j)$ ที่ $i < j$ และ $a_i < a_j$

(คือนับคู่ที่ "ตัวหน้าน้อยกว่าตัวหลัง")

### Input

```
N
a_1 a_2 ... a_N
```

### Output

- จำนวนคู่ที่ตรงเงื่อนไข

### ตัวอย่าง

```
Input:
5
1 10 3 5 7

Output:
7
```

**คำอธิบาย:**
คู่ที่ $a_i < a_j$ (ตัวหน้าน้อยกว่าตัวหลัง):
- (1,10), (1,3), (1,5), (1,7) → 4 คู่
- (3,5), (3,7) → 2 คู่
- (5,7) → 1 คู่
- รวม = **7** คู่

> 💡 **Hint:** Fenwick Tree (BIT) หรือ Merge Sort — ไล่จากซ้ายไปขวา ทุกครั้งที่เจอ $a_i$ ให้ query Fenwick Tree ว่ามีกี่ตัวที่น้อยกว่า $a_i$ อยู่ก่อนหน้าแล้ว (แต่โจทย์นี้นับ "น้อยกว่า" ไม่ใช่ inversion ปกติ — ต้องระวัง) ความซับซ้อน $O(N \log N)$

## ข้อ 14: Stacking (แพ็คกล่องลง 6×6)

### โจทย์

*(เหมือน UVa 311 - Packets)*

มีพัสดุรูปสี่เหลี่ยมจัตุรัส 6 ขนาด: $1 \times 1$, $2 \times 2$, $3 \times 3$, $4 \times 4$, $5 \times 5$, $6 \times 6$

กำหนดจำนวนพัสดุแต่ละขนาด ต้องใส่ลงใน **กล่องลังขนาด $6 \times 6$** ให้หมด

จงหา **จำนวนลังน้อยที่สุด** ที่ต้องใช้

### Input

```
a b c d e f
```

- $a$ = จำนวนพัสดุ $1 \times 1$
- $b$ = จำนวนพัสดุ $2 \times 2$
- $c$ = จำนวนพัสดุ $3 \times 3$
- $d$ = จำนวนพัสดุ $4 \times 4$
- $e$ = จำนวนพัสดุ $5 \times 5$
- $f$ = จำนวนพัสดุ $6 \times 6$

(input จบเมื่อทุกค่าเป็น 0)

### Output

- จำนวนลังน้อยที่สุด

### ตัวอย่าง

**ตัวอย่างที่ 1:**
```
Input:
0 0 4 0 0 0

Output:
1
```
(ของ 3×3 จำนวน 4 ชิ้น ใส่ลัง 6×6 ได้พอดี 1 ลัง)

**ตัวอย่างที่ 2:**
```
Input:
7 5 1 0 0 0

Output:
1
```

**คำอธิบาย:**
- ลัง 1: ใส่ 3×3 ได้ 1 ชิ้น → เหลือพื้นที่ 27 ช่อง
- ใส่ 2×2 ได้ 5 ชิ้น (ใช้ 20 ช่อง) → เหลือ 7 ช่อง
- ใส่ 1×1 ได้ 7 ชิ้น → พอดี!

**ตัวอย่างที่ 3:**
```
Input:
0 0 0 0 0 1

Output:
1
```
(ของ 6×6 ใส่พอดี 1 ลัง)

> 💡 **Hint:** Greedy — จัดของจากใหญ่ไปเล็ก:
> - ของ 6×6, 5×5, 4×4 → ใช้คนละลัง (1 ชิ้นต่อลัง)
> - ของ 3×3 → ใส่ได้ 4 ชิ้นต่อลัง
> - ของ 2×2 และ 1×1 → อุดตามช่องว่างที่เหลือจากลังที่มีของ 4×4, 5×5, 3×3
> - สุดท้ายถ้ายังมี 2×2 กับ 1×1 เหลือ → คำนวณจำนวนลังเพิ่มเติม
> - ต้องระวัง edge case เยอะ!
